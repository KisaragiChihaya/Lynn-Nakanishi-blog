笔记本电脑系统迁移小记

前言

相信大部分中国的电脑用户和我一样，从第一次接触电脑开始使用的电脑系统都是 Windows （mac 用户请当我没说过这句话）。诚然，微软是个伟大的公司，但是 Windows 却不是个伟大的系统，从中国网民最早接触到的 Windows 3.0，到小学电脑课上接触到的 Windows 95，再到风靡全国的 Windows XP，还有现在网吧最常见的 Windows 7……不管是哪个版本的 Windows，崩溃、死机、蓝屏都与用户的日常使用形影不离。而自从 某阿三 当上 CEO 之后更是砍掉了 Windows 测试部门转而发展 Azure，以至于从 Windows 10 开始，Windows 就像一团剪不断理还乱的乱麻，使用及其糟心。而我也是深受其折磨，却又一直抱着“又不是不能用”的心态继续凑合着用，但近日我的笔记本蓝牙模块频频故障，不论是卸载设备还是重装驱动都无法彻底解决问题，终于我忍无可忍，决定放弃 Windows 系统，转而投入 GNU/Linux 的怀抱。

硬件参数

型号：华硕 X450JB
CPU：Intel Core i5-4200H
独立显卡：NVIDIA GeForce 940M
内存：4GB DDR3L 1600 ×2
硬盘：500G SATA SSD

虽然硬件不咋地，但是它省电啊！我拆机之后才发现这笔记本的电池就只是 4 个 18650 电芯串联起来……

安装

系统选择

我平常在服务器上使用的都是 Debian，所以肯定是选用 Debian 来安装。至于为什么用 Debian 而不用更主流的 Ubuntu，是因为它是目前唯一一个还在使用 GNU/Linux 而不是 Linux 作为后缀的主流发行版 （虽然大部分的 Debian 都运行在非自由的固件上，而且 Debian 社群自己也在维护着非自由库）。但是选择桌面环境的时候我犯难了：我从没用过带桌面环境的 GNU/Linux 啊！Debian 支持 GNOME、KDE、LXDE、Xfce、Cinnamon 还有 MATE 环境，这让本选择恐惧症患者左右为难。我把所有的桌面环境都装了一遍之后， 想来想去，还是选择了 KDE 桌面环境，主要是因为它相对于 GNOME 桌面环境来说拥有 Windows 用户熟悉的设计风格，更容易上手，同时也比其他的桌面环境可定义性要高。

正式安装

在 下载页面 下载好网络安装版镜像之后使用 Rufus 或者 balenaEtcher 烧录进移动存储设备中，然后不拔除设备重启，在开机界面狂按 ESC 键选择设备就能进入安装界面了。

具体安装过程忘记截图了，不过全程都是图形界面，十分简单易懂。而且顺带一提 Live CD 版使用的是 Calamares 安装程序，代替了 Debian 安装程序，对普通用户更友好。

部分预装软件

Firefox ESR（延长支持版）

Debian 默认只有 ESR 版本可以安装，当然你也可以使用其他方式（如 Flatpak 和 Snap ）安装最新版本的 Firefox。

Konqueror

默认的网页浏览器，因为不能装扩展，所以我还是乖乖滚来用 Firefox 了。

LibreOffice Writer

个人感觉在几个自由的办公套件中 LibreOffice 算是好用的一个。

Kate

默认的文本编辑器，支持大多数编程语言 / 标记语言的语法高亮。

Juk

默认的音乐播放器，功能挺全。

KDE Connect

超级好用的小工具，可以多设备之间互相遥控、远程输入、发送文件、同步通知……等等等等。
还支持 人间之屑 Windows、macOS、Android、SailfishOS 和 Plasma Mobile。

GNU 图像处理工具（GIMP）

大名鼎鼎的 GIMP，整体体验和 Windows 上的版本没有多少区别。
至于为什么没有预装 Krita，个人觉得是因为 Krita 主打数字绘画，并不能像 GIMP 一样作为通用图片编辑器使用，所以才没有预装。

软件推荐

GNU/Linux 下也有很多好用的软件，在这里推荐一些。

排名不分先后。

VLC

官方网站：www.videolan.org/vlc/

支持平台：GNU/Linux（Debian GNU/Linux · Ubuntu · Mint · openSUSE · Gentoo Linux · Fedora · Arch Linux · Slackware Linux · Red Hat Enterprise Linux）、BSD（FreeBSD · NetBSD · OpenBSD）、Windows（包括 Windows Phone）、macOS、Chrome OS、Solaris、QNX、Syllable Desktop、OS/2、Android、iOS

许可协议：GNU GPLv2 或更新版本（播放器）、GNU LGPLv2.1 或更新版本（引擎）

无需多言，可能是支持平台最多的媒体播放器。

Celluloid

官方网站：celluloid-player.github.io

支持平台：GNU/Linux（Debian GNU/Linux · Arch Linux · Fedora · Frugalware · OpenSUSE · Gentoo Linux · Guix · Solus · Ubuntu）

许可协议：GNU GPLv3

原名 GNOME MPV，其实就是给 MPV 加了个 GTK 的壳，不过这样一来使用就很方便了。

Kaffeine

官方网站：apps.kde.org/kaffeine/

支持平台：GNU/Linux

许可协议：GNU GPLv2 或更新版本

和 KDE 环境下预装的 Dragon Player 同属 KDE 开发，但是 Kaffeine 更像是光盘播放器，甚至可以播放数字电视 （前提是你要有源）。

Kdenlive

官方网站：kdenlive.org

支持平台：GNU/Linux、FreeBSD、Windows、macOS

许可协议：GNU GPLv2 或更新版本

我一直在用的剪辑软件，导出剪辑时调用的是 ffmpeg，比 Pr 不知高到哪里去了。

Aegisub

官方网站：aegisub.org

支持平台：GNU/Linux、FreeBSD、Windows、macOS

许可协议：3-clause BSD License

这个不用过多介绍了，只要做过字幕的都听说过。

Krita

官方网站：krita.org

支持平台：GNU/Linux、FreeBSD、Windows、macOS、Chrome OS、Android

许可协议：GPLv3

前面说 GIMP 的时候已经提到了 Krita，其实 Krita 也是我一直在用的数字绘画软件。作为数字绘画软件中为数不多的自由软件，Krita 只凭“自由”这一点就已经做得很好了。而软件本身功能也很完善，甚至还有日本动画专用的模板，这点必须点赞！不过我画技实在太烂，还是不放绘图过程的截图了 （事实证明一个好工具也不能改变自己菜的事实）。但有一点需注意：如果直接从自带的软件商店 Discover 里安装的话默认是没有中文的，需要补充安装 krita-l10n 这个包。

这样就能在设置中选择中文了。

后记

对于大多数人来说，碰到和我一样的问题的时候一般会选择把蓝牙驱动修好，而不是像我一样选择更换操作系统。但对于我来说，这次事件更像是导火索：Windows 封闭的环境、微软放置不管的态度使得 Windows 光是让普通用户日常使用就叫苦不迭，那与其再继续忍受难用的 Windows，为什么不选择拥抱自由开放的 GNU/Linux 呢？
